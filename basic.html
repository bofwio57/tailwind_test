tailwind 대해

장점

UI 개발 생산성을 획기적으로 높여주는 핵심

단점

HTML 코드의 가독성이 떨어짐

기본

단위체계

                기본 단위(예: w-4)는 rem을 기반
                고정된 크기가 필요하다면 대괄호를 사용하여 `w-[20px]`와 같이 임의의 고정값(Arbitrary values)을 직접 지정할 수 있습

색상, 테두리, 라운드 처리

                백그라운드 색상은 `bg-색상-숫자` 형태로 지정하며 숫자가 커질수록 색상의 명도와 채도가 짙어지는 구조
                bg, border, border-color, rounded
                bg-pink-400 border-8 border-black rounded-[25px] rounded-full>동그라미

레이아웃: 너비(W), 높이(H), 패딩(P), 마진(M)

                width(W), height(H), padding(P), margin(M)
                mx-10  x축(좌우)과 y축(상하)의 마진 및 패딩을 별도로 제어하는 것이 가능
                w-full 화면 전체

텍스트

                text-color, text-size, font-bold, cursor
                마우스 커서 모양을 변경하려면 `cursor-pointer`
                text-pink-400 text-2xl text-[50px] font-[100]

flex

                flex, flex-col, justify, items, gap-4
                flex justify-center items-center > 정 가운데 정렬

호버(Hover)

                `hover:`
                hover:bg-purple-400 transition >호버시 백그라운드가 퍼블로 변해라 자연스럽게(transition)

고급 기능: 그룹 호버(Group Hover)

                `group-hover`를 적용하면, 상위 요소에 마우스를 올렸을 때 하위 요소의 스타일이 동시에 변경
                최상위 요소에 group 부여
                스타일 변경하는 하위 요소에 group-hover:~ 부여 하면 변경됨
